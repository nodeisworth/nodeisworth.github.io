So from the above example we see that ProductService communicates with BillService using the service api but there are disadvantages in using this method.

Disadvantages of using service API :

  1. High coupled communication between services for example if we changed the endpoint name in BillService from getInvoice to getInvoiceModified we should change it also in the product service that calls it

  2. Not stable communication as for example if billservice went offline so request will fail and productservice will have to resend it

  3. Don't support complex routing for example if we want to send one request from product service to multiple services which is called broadtcasting , this would be hard and require more effort



For the disadvantages mentioned above comes the importance of rabbitmq which is a message broker that allows low coupled communication between multiple components and also supports stable communication and supports complex routing and that what we will discuss in the next lesson
